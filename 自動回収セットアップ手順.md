# 自動回収セットアップ手順

この構成では、GitHub Pagesに置いた学習ログアプリから、Google Apps Scriptを通してGoogleスプレッドシートにログを自動保存します。

## 使うもの

- GitHub Pages: 対象者が開くアプリ置き場
- Googleスプレッドシート: ログの保存先
- Google Apps Script: アプリから送られたログをスプレッドシートに書き込む受け口

## 1. Googleスプレッドシートを作る

1. Google Driveで新しいスプレッドシートを作る
2. 名前を例として「学習ログ自動回収」にする
3. そのスプレッドシートで「拡張機能」→「Apps Script」を開く

## 2. Apps Scriptにコードを貼る

`google-apps-script-code.gs` の中身を、Apps Scriptの `Code.gs` に貼り付けます。

貼り付けたら保存します。

## 3. Webアプリとして公開する

Apps Script画面で次のように設定します。

1. 右上の「デプロイ」→「新しいデプロイ」
2. 種類は「ウェブアプリ」
3. 実行ユーザーは「自分」
4. アクセスできるユーザーは「全員」
5. デプロイする
6. 表示されたウェブアプリURLをコピーする

このURLは `https://script.google.com/macros/s/.../exec` のような形です。

## 4. アプリ側にURLを貼る

`public/collector-config.js` を開いて、コピーしたURLを貼ります。

```js
window.LEARNING_LOG_COLLECTOR_URL = "https://script.google.com/macros/s/ここにURL/exec";
```

## 5. GitHub Pagesに置く

最低限、次の3ファイルを同じ場所に置きます。

```text
endurance-version.html
endurance-version.js
collector-config.js
```

GitHub Pagesで `endurance-version.html` を開けるようにします。

## 6. テストする

1. GitHub PagesのURLを開く
2. 参加者IDを `TEST001` などにする
3. 2〜3問だけ解く
4. 終了する
5. Googleスプレッドシートに `sessions` と `logs` のシートが作られているか確認する

## 注意

- 対象者の本名は入れず、`P001` のような参加者IDだけにする
- 自動回収が失敗した時のため、終了画面のCSV保存も残しておく
- 実験前に、保護者・本人・支援者へ「操作ログを取得する」ことを説明する
